terraform-azure-service-bus

Executive Summary

This Terraform module deploys a production-ready Azure Service Bus namespace with comprehensive messaging infrastructure. It supports queues with dead-letter handling, topics with subscription filtering (SQL and correlation), multi-level authorization rules, network rule sets with IP and VNet controls, private endpoint connectivity, managed identity, customer-managed key encryption, zone redundancy, and diagnostic settings. The module enables enterprise-grade asynchronous messaging for microservices, event-driven architectures, and distributed systems.

Component Breakdown

Service Bus Namespace

Core messaging namespace with configurable SKU (Basic/Standard/Premium), capacity (1-16 messaging units for Premium), zone redundancy, managed identity, CMK encryption, and TLS enforcement.

Queues

Point-to-point messaging with max delivery count, lock duration, message TTL, auto-delete, duplicate detection, batched operations, express mode, partitioning, session support, and dead-letter on expiration. Supports message forwarding.

Topics and Subscriptions

Publish-subscribe messaging. Topics support ordering, partitioning, and duplicate detection. Subscriptions filter messages via SQL or correlation filters with configurable dead-lettering and forwarding.

Authorization Rules

Granular access control at namespace, queue, and topic levels with Listen, Send, and Manage permissions. Each rule generates independent connection strings and access keys.

Network Security

Network rule sets with default action, trusted services bypass, IP rules, and VNet subnet rules. Private endpoints provide secure connectivity with DNS zone group integration.

Diagnostics

Configurable diagnostic settings supporting Log Analytics, Storage Account, Event Hub, and partner solutions. Tracks OperationalLogs, VNetAndIPFilteringLogs, RuntimeAuditLogs, and ApplicationMetricsLogs.

Data Flow

  1. Producers authenticate via namespace/queue/topic authorization rules or managed identity
  2. Messages are sent to queues (point-to-point) or topics (pub-sub)
  3. For topics, subscriptions filter messages using SQL or correlation filters
  4. Consumers receive and process messages with configurable lock duration
  5. Failed messages are retried up to max_delivery_count, then dead-lettered
  6. Network rules and private endpoints control network-level access
  7. Diagnostic settings stream operational and audit logs to monitoring targets

Industry Adaptation

Microservices

Use queues for command patterns, topics for event broadcasting. Enable sessions for ordered processing and duplicate detection for idempotency.

E-Commerce

Order processing queues with dead-letter handling, event topics for inventory/shipping notifications, Premium SKU for guaranteed throughput.

Financial Services

Premium SKU with CMK encryption, private endpoints, disabled local auth, zone redundancy for HA, full audit logging enabled.

IoT / Telemetry

High-throughput topics with partitioning, batched operations, subscription filters for routing telemetry to different processing pipelines.

Production Readiness Checklist

  • Select appropriate SKU tier (Premium for production workloads)
  • Configure zone redundancy for Premium namespaces
  • Set minimum TLS version to 1.2
  • Disable local auth and use managed identity where possible
  • Configure network rules to restrict public access
  • Deploy private endpoints for secure connectivity
  • Enable dead-letter queues for failed message handling
  • Set appropriate max_delivery_count for retry policies
  • Configure diagnostic settings for operational monitoring
  • Set up authorization rules with least-privilege access
  • Enable customer-managed key encryption for compliance
  • Apply resource tags for governance and cost tracking
